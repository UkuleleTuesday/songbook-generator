Experimental: Drive-based editions

These editions are defined by a .songbook.yaml file inside a Google Drive folder. Each folder that is a direct child of the configured source folders and contains a valid .songbook.yaml will appear here. Click the folder_open icon next to an edition to open its Drive folder.

How it works:

  • The .songbook.yaml uses the same schema as config-based editions — it defines the title, description, and song selection filters.
  • Cover, preface, and postface pages can be auto-detected from Cover/, Preface/, and Postface/ subfolders by setting use_folder_components: true in the YAML.
  • Drive shortcuts are fully supported — use them to include files from other folders without duplicating them.
  • Folders without a valid .songbook.yaml are silently skipped.

Because this feature reads live Drive content it may behave differently from config-based editions and is still under active development.
